# Project Brief: AI Presentation Studio

## Objective

Build a clean browser website where users can generate, edit, preview, and export editable PowerPoint presentations. AI integration is important, but the app must also work without AI so users are never blocked by missing keys.

## Current Scope

This is a practical MVP focused on the core workflow:

1. Add a Claude API key or choose no-AI mode.
2. Generate a deck from a prompt.
3. Edit slide content and design in the browser.
4. Preview the deck.
5. Download a native `.pptx` file.
6. Save or load editable project JSON when needed.

## Core Features

- Prompt-to-PPT generation with Claude
- Claude model selection using current generic API IDs
- No-AI editable template generation
- Single API key field for users
- Server `.env` key fallback with `CLAUDE_API_KEY`
- Help Me section for first-time users
- Slide preview modal
- Slide move controls for basic reordering
- Editable slide inspector
- Multiple slide layouts and color themes
- Optional local slide visual creation from each visual prompt
- Server-side PowerPoint export
- Browser-side PowerPoint export fallback
- Editable project save/load

## UX Requirements

- The generate button should say **Generate PPT**.
- API settings should only ask for a Claude API key.
- The selected Claude model should be visible and editable.
- No provider or image model fields should appear in the UI.
- The app should automatically use the saved browser key first, then the local/server key.
- The app must remain usable without an API key.
- **Download PowerPoint** should produce `.pptx`, not JSON.
- JSON should only be downloaded through **Save Editable Project**.
- Users should be able to edit generated decks immediately.

## Architecture

Frontend:

- Plain HTML, CSS, and JavaScript
- Local storage for the user API key
- Live slide editor and preview
- Browser-side PPTX fallback via `vendor/pptxgen.bundle.js`

Backend:

- Node.js HTTP server
- Static file serving
- `/api/generate-deck`
- `/api/generate-image`
- `/api/export-pptx`
- Vercel serverless wrappers in `api/`

PowerPoint:

- Server export uses `pptxgenjs`
- Browser fallback uses bundled `pptxgenjs`

## Vercel and Supabase Direction

Vercel is the preferred public hosting target for this version:

- Static app from the repository root
- Serverless functions from `api/`
- `CLAUDE_API_KEY` stored as a Vercel environment variable

Supabase should be added when the app needs persistent product features:

- Auth
- Saved decks
- Project history
- Team libraries

Initial Supabase environment placeholders:

```bash
SUPABASE_URL=your_supabase_project_url
SUPABASE_ANON_KEY=your_supabase_anon_key
```

## Known Limits

- No database or user accounts yet.
- No cloud persistence yet.
- Custom API keys are saved in browser local storage only.
- Slide visual creation is local and does not require a separate image model key.
- Browser-side PPTX export is a fallback and may be simpler than server-side export.

## Recommended Next Steps

- Deploy the website on a Node-friendly host such as Render, Railway, Fly.io, or a similar service.
- Add authentication if this becomes a public product.
- Add persistent projects with a database.
- Add a template gallery and reusable brand kits.
- Add drag-and-drop slide reordering.
- Add brand ingestion: logo upload, brand colors, fonts, CSS variables, and reusable brand presets.
- Add source document upload, extraction, and indexing for research-heavy decks.
- Explore stock image/MCP asset providers and specialized image generators for slide visuals.
